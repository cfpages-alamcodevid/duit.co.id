# 2. Core Angle
Laba tidak sama dengan kas. Artikel harus mengajari pemilik UMKM membaca arus kas operasional harian dan memutuskan prioritas pembayaran.

# 3. Indonesia Context
- Banyak UMKM menghadapi mismatch tempo: supplier minta cepat, pelanggan bayar lambat.
- Penggunaan QRIS/e-wallet mempermudah transaksi tapi perlu disiplin settlement reconciliation.
- Akses pembiayaan formal tergantung kualitas catatan keuangan.

Angka acuan:
- Buffer kas sehat 2-3 bulan biaya tetap (Perlu verifikasi).
- DSO (days sales outstanding) yang terlalu panjang mengganggu siklus belanja stok.

# 4. Example Numbers
Kasus sederhana:
- Penjualan bulanan Rp120 juta
- Margin kotor 35%
- Biaya tetap Rp28 juta
- Piutang rata-rata 45 hari

Masalah:
- Secara laba terlihat positif, tapi kas minus saat jatuh tempo gaji/supplier.

Perbaikan:
- Minta DP 30-50% untuk order custom.
- Negosiasi tempo supplier +15 hari.
- Program diskon pembayaran cepat ke pelanggan B2B.

# 5. Risks
- Menggabungkan kas pribadi dan bisnis.
- Belanja aset dari kas operasional.
- Tidak punya proyeksi kas mingguan.

Kontrol:
- Rekening operasional terpisah.
- Forecast kas 13 minggu.
- Prioritas pembayaran: payroll, supplier kritis, kewajiban pajak.

# 6. Persona Scenarios
Persona A: Konveksi kecil, order musiman.
Fokus: dana penyangga dan skenario low-season.

Persona B: Event organizer, pembayaran klien sering mundur.
Fokus: kontrak termin pembayaran yang tegas.

# 7. Myths to Debunk
- "Selama untung, bisnis aman." Tidak, kehabisan kas bisa menutup usaha.
- "Piutang pasti cair tepat waktu." Tidak, harus ada buffer.
- "Catatan kas cukup diingat." Tidak, wajib sistem tertulis.

# 8. Action-Step Outline
1. Pisahkan kas bisnis-pribadi.
2. Buat laporan arus kas sederhana mingguan.
3. Monitor DSO, DPO, dan persediaan.
4. Percepat penerimaan kas dan tunda pengeluaran non-kritis.
5. Bentuk buffer kas minimum.
6. Review forecast tiap minggu.
