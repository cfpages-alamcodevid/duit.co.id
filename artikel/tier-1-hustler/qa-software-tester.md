## Apa Itu QA Tester dan Software Tester?

QA Tester itu orang yang jaga kualitas software. Kalau ada bug, error, atau fitur nggak jalan, tugas dia cari dan laporin.

Software Tester hampir sama. Bedanya cuma di fokus: QA lebih ke proses, tester lebih ke produk. Tapi di Indonesia, 90% perusahaan sebutnya sama aja.

Jadi jangan bingung kalau lihat lowongan dengan dua judul itu. Isinya sama kok.

Gampangnya: orang yang pastiin aplikasi yang kamu pake nggak error pas digunakan.

## Skill Apa yang Harus Dikuasai?

Skill wajib nggak sesulit developer kok. Nggak perlu jago coding sampai bikin aplikasi sendiri.

### Hard Skills
1. Manual testing: Cek fitur software secara manual, catat bug yang ditemukan.
2. Automation basics: Selenium buat web testing, Postman buat API testing. Nggak perlu jago coding banget, cuma dasar aja.
3. Bug reporting: Pakai Jira atau Trello, tulis laporan bug yang jelas dan lengkap.
4. SQL basics: Query data buat cek hasil testing di database.

### Soft Skills
1. Detail-oriented: Jeli lihat error kecil yang nggak kelihatan orang lain.
2. Komunikasi: Bisa jelasin bug ke developer dengan jelas tanpa emosi.

Tahu nggak? 80% QA pemula gagal karena underestimate soft skills. Bukan karena nggak bisa pakai tools.

Jangan langsung belajar automation testing dulu. Mulai dari manual testing, baru naik ke automation kalau sudah paham dasarnya.

## Cara Mulai Karir QA Tester Tanpa Background IT

Langkah yang saya sarankan buat pemula, tanpa perlu gelar IT:

### 1. Pelajari Dasar QA Testing
- Ikuti course gratis di YouTube: cari "QA testing untuk pemula Indonesia".
- Belajar manual testing dulu, jangan langsung automation.
- Waktu: 2-4 minggu (1 jam per hari cukup).

### 2. Pilih Tools Sesuai Budget
- Gratis: Selenium, Postman, Jira (free tier), Trello.
- Berbayar: UFT, TestComplete (buat senior, nggak perlu buat pemula).
- Pemula cukup yang gratis dulu, nanti kantor yang sediain kalau sudah kerja.

### 3. Build Portfolio
- Testing project open source di GitHub, taruh laporan bug sampel.
- Buat akun LinkedIn, posting hasil testing project sampel.
- Portofolio lebih penting dari gelar IT. Saya lihat sendiri banyak yang lulusan SMA diterima cuma karena portofolio oke.

### 4. Cari Klien/Lowongan
- Platform freelance: Freelancer.com, Upwork, Fiverr. Mulai dari rate rendah dulu buat dapet review.
- LinkedIn: Follow komunitas QA Indonesia, apply lowongan entry-level.
- Jangan terima project tanpa kontrak tertulis. Ini kesalahan fatal banyak pemula.

**⚠️ Peringatan:**

{/callout}

## Metodologi Testing yang Perlu Dikuasai

Jangan cuma klik-klik aja. Testing itu ada metodologinya:

1. **Black Box Testing**: Cek fitur dari sisi user, nggak perlu tahu kode program. Cocok buat pemula.
2. **White Box Testing**: Cek kode program, butuh skill coding. Buat senior.
3. **Agile Testing**: Testing di metodologi kerja agile, sekarang 80% perusahaan pakai ini.
4. **Regression Testing**: Cek apakah fitur lama masih jalan setelah fitur baru ditambah.

Saya perhatikan kebanyakan lowongan entry-level cuma butuh Black Box dan Agile Testing dasar. Nggak perlu pusing belajar yang susah-susah dulu.

## Kesalahan Umum Pemula QA Tester

90% pemula gagal karena 2 kesalahan ini, jangan sampai jadi korban:

1. **Underestimate testing**: Kira cuma klik-klik aja. Padahal butuh metodologi sistematis dan dokumentasi rapi.
2. **Underprice diri sendiri**: Minta Rp 20.000/jam. Padahal rate pasar pemula Rp 50.000/jam.

Kesalahan lain: nggak bikin kontrak tertulis. Saya baca di Reddit r/indonesia banyak yang nggak dibayar klien karena nggak ada bukti kontrak.
