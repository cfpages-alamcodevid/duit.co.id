## Purpose

Extra adalah bonus pelengkap artikel yang dipakai untuk mengubah pembaca menjadi member. Bentuk idealnya PDF kecil, satu sampai tiga halaman, yang dibuat client-side dari file Markdown saat user menekan tombol download. ## Download Mechanism

- Artikel menampilkan tombol download extra jika file `/extra/{tier}/{slug}-extra.md` tersedia.
- Saat tombol diklik, tampilkan modal viewport-fixed di tengah layar, bukan relatif terhadap parent/container.
- Jika user belum login, modal menampilkan login/register tabs Clerk dan Google sign-in.
- Setelah login, Markdown extra dirender dengan CSS Duit.co.id lalu dikonversi menjadi PDF client-side.
- Tidak ada PDF statis yang disimpan. Yang disimpan hanya Markdown extra dan event analytics/download jika perlu.

## Priority

- P0: extra berdampak langsung untuk kondisi krisis, hukum, atau tindakan cepat.

- P2: extra membantu keputusan investasi, properti, legal, atau sistem bisnis.
- P3: extra untuk topik UHNW/legacy yang lebih cocok sebagai memo keluarga/advisor.

## Quality Rules

- Extra harus melengkapi artikel, bukan mengulang isi artikel.
- Target 500-1.200 kata, kecuali direktori yang membutuhkan tabel lebih panjang.
- Gunakan tabel, checklist, worksheet, skrip, atau template yang bisa langsung dipakai.
- Jangan pakai H1 di body. Title dari frontmatter adalah H1.
- Jangan tulis copy internal seperti mockup, sandbox, backlog, atau catatan untuk founder.
- Gunakan angka Rupiah lengkap: `Rp 1 juta`, bukan `Rp 1M` atau `1jt`.
- Untuk direktori/link eksternal, sertakan URL, fungsi, kelebihan, kekurangan, dan catatan biaya/risiko.
